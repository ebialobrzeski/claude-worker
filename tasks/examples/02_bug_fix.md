# Fix one bug

Investigate this codebase for bugs — logic errors, incorrect error handling,
off-by-one errors, race conditions, or incorrect assumptions.

Steps:
1. Read the code and look for places where behavior likely diverges from intent.
2. Pick the single clearest, most impactful bug you find.
3. Fix it. Do not change unrelated code.
4. Explain what the bug was, why it matters, and what you changed.

If no bugs are found, say so and make no changes.
